# 个人博客系统需求文档（Vue + Spring Boot + MyBatis）

---

## 1. 项目概述

### 1.1 项目背景
独立博客是知识沉淀、技术分享、个人品牌塑造的重要平台。为了完全掌控内容与数据，需要基于主流前后端分离技术栈构建一个自主可控的博客系统。

### 1.2 技术约束
- **前端**：Vue 3 + Vite + Vue Router + Pinia
- **后端**：Spring Boot 2.7+ / 3.x + MyBatis / MyBatis-Plus
- **数据库**：MySQL 8.0（或 MariaDB）
- **认证**：Spring Security + JWT
- **部署**：前端 Nginx / 静态托管，后端 JAR 包部署于 Linux 服务器或 Docker 容器
- **其他**：遵循 RESTful API 规范，前后端完全分离，通信基于 JSON

---

## 2. 用户角色与场景

| 角色 | 描述 |
|------|------|
| **博主** | 唯一后台管理者，负责内容创作、评论管理、站点配置。 |
| **读者** | 浏览博客、搜索文章、发表评论、订阅 RSS。 |

### 典型场景
1. 博主通过后台 Markdown 编辑器创作文章，上传封面图至服务器或图床，设置分类、标签后发布。
2. 读者访问首页看到文章列表，点击阅读详情，享受代码高亮与数学公式渲染，文末可发表评论。
3. 读者通过分类、标签、时间线筛选文章，通过搜索框快速定位内容。
4. 博主审核评论，回复读者，管理文件（图片/附件），通过仪表盘概览博客数据。

## 3. 功能性需求

### 3.1 前台展示（读者端）

#### 3.1.1 通用布局
- **顶部导航**：博客名称、首页、归档、分类、标签、关于、搜索入口
- **底部信息**：版权声明、社交链接、备案号、主题切换按钮
- **响应式**：移动端自动折叠菜单为汉堡图标，内容区域自适应宽度
- **暗色模式**：基于系统偏好或手动切换，持久化到 localStorage

#### 3.1.2 首页
- **文章列表**：倒序分页展示（后端分页），每项包含封面图（可缺省）、标题、摘要（自动生成或手动填写）、发布日期、分类名、标签列表
- **置顶文章**：支持最多 3 篇置顶，特殊标识展示在列表顶部
- **侧边栏区（可选）**：个人简介卡片、分类列表、标签云、最近文章、归档链接

#### 3.1.3 文章详情
- **Markdown 渲染**：通过 `markdown-it` 或 `marked` + `highlight.js` 实现代码语法高亮，支持 GFM 表格、任务列表、Emoji
- **数学公式**：集成 KaTeX 或 MathJax，自动渲染块级与内联公式
- **目录导航**：自动抽取 h1~h3 标题生成锚点目录，侧边吸附滚动高亮当前标题
- **图片预览**：点击放大，支持灯箱效果
- **版权声明**：文末自动展示自定义协议（如 CC BY-NC-SA）
- **上下文导航**：文章底部显示同一分类下的上一篇/下一篇标题与链接
- **元数据展示**：作者、发布时间、分类、标签、阅读量
- **SEO 元信息**：动态 `<title>`、`<meta description>`、Open Graph 标签，服务端可配合生成静态页（可选）

#### 3.1.4 归档/分类/标签
- **按时间归档**：按年-月分组展示文章列表，点击可跳转到该月文章
- **分类页**：列出所有分类及文章数量，点击分类展示该分类下文章列表（分页）
- **标签页**：标签云（字号随文章数量变化），点击标签展示对应文章列表（分页）

#### 3.1.5 搜索功能
- **关键词搜索**：搜索文章标题和内容（后端 LIKE 查询或全文索引），高亮匹配词
- **搜索结果页**：分页展示结果，无结果时给予提示

#### 3.1.6 评论系统
- **发表评论**：填写昵称（必填）、邮箱（必填）、网址（选填），内容支持 Markdown 基础语法（加粗、代码等），可输入验证码或人机验证（如图形验证码）
- **评论展示**：嵌套回复（两级），头像通过 Gravatar 获取，内容 Markdown 渲染，时间格式化为“多少分钟前”
- **审核机制**：评论提交后默认进入待审核状态，前台仅显示已审核通过的评论，并提示“评论审核后可见”

#### 3.1.7 其他
- **RSS 订阅**：提供 RSS 2.0 / Atom 订阅链接，输出最近文章摘要
- **站点地图**：动态生成 sitemap.xml（可选阶段实现）
- **自定义页面**：如“关于我”页面，后台可编辑 Markdown 内容并独立展示（V1.1+）

---

### 3.2 后台管理（博主端）

#### 3.2.1 认证与账户
- **单用户登录**：用户名 + 密码登录，密码 bcrypt 加密存储
- **JWT 鉴权**：后端签发 Access Token（有效期可配），前端拦截器自动携带；可根据需要加入 Refresh Token
- **登录保护**：连续失败 5 次锁定 30 分钟，记录登录日志
- **简单个人设置**：可修改密码、头像、昵称（可选）

#### 3.2.2 文章管理
- **文章列表**：表格展示标题、状态（已发布/草稿）、分类、标签、创建时间、更新时间；支持按状态筛选、排序，支持模糊搜索标题
- **新建/编辑文章**：
    - Markdown 编辑器（如 @toast-ui/editor 或 mavon-editor），支持实时预览、全屏、分屏编辑
    - 上传封面图（支持本地存储或图床上传）
    - 选择分类（单选）、标签（多选，可即时新建标签）
    - 自定义文章 slug（用于友好 URL）
    - 摘要：可手动填写，也可从内容自动截取
    - SEO 设置：自定义 title、description、关键词
    - 草稿与发布状态切换
    - 预览按钮（新窗口打开预览页面）
- **批量操作**：批量删除、批量修改分类/标签（可选）
- **回收站**：软删除文章，可恢复或永久删除

#### 3.2.3 分类管理
- 字段：分类名称、slug、描述（可选）、创建时间
- 增、删、改、查，删除前校验是否有文章关联

#### 3.2.4 标签管理
- 字段：标签名称、slug、创建时间
- 增、删、改、查，支持合并标签

#### 3.2.5 评论管理
- **待审核列表**：展示待审核评论，可单条通过/拒绝，或批量操作
- **已审核列表**：所有通过审核的评论，可回复、标记为垃圾、删除
- **垃圾箱**：被标记为垃圾的评论列表，可恢复或永久删除
- **管理员回复**：直接在前台评论下显示管理员回复，与普通评论样式区分

#### 3.2.6 媒体管理（文件库）
- **上传接口**：支持图片、附件上传，生成可访问 URL
- **文件列表**：缩略图或文件名展示，支持复制 Markdown 图片格式、图片链接，可删除
- **存储方式**：文件保存在服务器本地磁盘或集成对象存储（阿里云 OSS / MinIO），初期可配置本地路径

#### 3.2.7 站点配置
- **基本设置**：网站标题、副标题、站点描述、Logo URL、Favicon URL、关于页面 Markdown 内容
- **社交链接**：GitHub、Twitter、Email、RSS 链接等，可添加/删除
- **SEO 全局**：首页 keywords、description、Google Analytics ID、百度统计 ID、robots.txt 配置
- **导航菜单管理**：自定义顶部或底部导航菜单项（名称、链接、打开方式、排序）
- **版权设置**：全局版权信息，支持 HTML 片段

#### 3.2.8 数据统计（V1.2+）
- **文章阅读量**：每篇文章的 PV，简单自实现（防刷，IP 去重简单方案）
- **仪表盘概览**：文章总数、分类数、标签数、评论总数、近期访问趋势简图（可后期集成）

## 4. 非功能性需求

### 4.1 性能指标
- 静态资源开启 Gzip/Brotli 压缩，前端资源使用 CDN 或 Nginx 缓存
- 首页文章列表 API 响应时间控制在 200ms 内（本地网络）
- 图片懒加载、代码块按需高亮，减少首屏包体积
- 后端合理使用 MyBatis 二级缓存、数据库连接池（HikariCP）

### 4.2 安全性
- **XSS 防护**：前端渲染用户输入时使用 DOMPurify 清洗；后端存储前可再次过滤
- **SQL 注入**：MyBatis 使用 `#{}` 预编译，严禁 `${}` 拼接用户输入
- **CSRF**：前后端分离通过 JWT + SameSite Cookie 可缓解，Token 不存 Cookie，前端手动在 Authorization 头携带
- **文件上传安全**：限制文件类型（白名单）、大小，重命名文件，禁止上传可执行脚本
- **验证码**：登录接口可选加图形验证码（如 EasyCaptcha），评论模块添加人机校验（简单后端生成的数学题验证码）

### 4.3 可维护性
- **后端**：分层清晰（controller / service / mapper），统一异常处理、统一响应体 `{ code, message, data }`
- **数据库**：规范命名，使用 Flyway 或 Liquibase 管理 SQL 迁移
- **API 文档**：Knife4j / SpringDoc 自动生成 API 文档，方便前端对接
- **前端**：组件化开发，目录结构按功能模块划分，状态管理集中

### 4.4 SEO 与可访问性
- 虽然前后端分离对 SEO 不利，但可通过以下方式弥补：
    - 使用 `prerender-spa-plugin` 预渲染首页和文章详情页（构建时生成静态 HTML）
    - 或引入服务端渲染（Nuxt.js 成本高，不采用），改用预渲染方案
    - 每个页面动态设置 meta 标签（通过 vue-meta 或 unhead）
- 语义化 HTML 标签，图片自带 alt 属性，键盘可访问

### 4.5 兼容性
- 支持主流浏览器最近两个版本（Chrome、Firefox、Safari、Edge）
- 移动端适配至 320px 宽度

## 5. 系统架构与接口规范

### 5.1 前后端交互
- 统一 Base URL，如 `/api/v1`
- 鉴权接口：`POST /api/v1/auth/login` 返回 `{ token }`
- 数据格式：`{ "code": 200, "message": "success", "data": {} }`
- 分页参数：`page`, `size`，返回 `{ total, list }`
- 错误码规范：业务错误码与 HTTP 状态码配合，如 401 未认证、403 无权限、404 不存在、422 参数校验失败

### 5.2 核心 API 列表（示例）
- 文章：`GET /api/v1/articles`（公开，分页+筛选），`GET /api/v1/articles/{slug}`（详情），`POST /api/v1/admin/articles`（新建），`PUT /api/v1/admin/articles/{id}`，`DELETE /api/v1/admin/articles/{id}`
- 分类/标签：公开查询与后台 CRUD 类似结构
- 评论：`GET /api/v1/articles/{slug}/comments`（公开已审核），`POST /api/v1/articles/{slug}/comments`（提交），后台 `/api/v1/admin/comments` 系列
- 文件：`POST /api/v1/admin/files/upload`

### 5.3 数据库设计要点
- 核心表：`article`（id, title, slug, summary, content_md, content_html, cover_url, status, category_id, is_top, create_time, update_time）、`category`、`tag`、`article_tag`、`comment`（pid 字段支持嵌套）、`user`、`site_config`（key-value 结构）、`media`
- 索引：`slug` 唯一索引、`category_id`、`status`、`create_time` 索引；`comment` 的 `article_id` 索引

## 6. 界面设计指导

- **前台风格**：极简主义，以阅读体验为核心，首页头部可包含博主大图与简介，文章列表采用卡片式或列表式
- **文章阅读页**：居中单栏，最大宽度 750px，字体 16-18px，行高 1.8，背景柔和
- **后台风格**：左侧深色导航，右侧内容区，使用 Element Plus 或 Ant Design Vue 组件库提高开发效率

## 8. 附录：开发规范建议

- 前端代码统一使用 ESLint + Prettier，提交前自动格式化
- 后端遵循阿里巴巴 Java 开发手册，使用 Maven 管理依赖
- Git 分支策略：`main`（正式）、`develop`（开发）、`feature/*`
- 环境配置分离：`application-dev.yml`、`application-prod.yml`，敏感信息加密
